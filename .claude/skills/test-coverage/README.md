# Test Coverage Skill

Evaluates test coverage for the current branch/PR, enforces thresholds, and reports metrics.

## Features

- **Auto-detects test framework**: Jest, Vitest, pytest, Go
- **Full or diff-only coverage**: Check entire codebase or just changed lines
- **Threshold enforcement**: Soft warnings or hard failures
- **PR integration**: Posts coverage reports as PR comments
- **Memory tracking**: Stores metrics for trend analysis
- **Multi-language support**: JavaScript/TypeScript, Python, Go

## Quick Start

```bash
cd /path/to/your/repo

# Check overall coverage
python3 .claude/skills/test-coverage/scripts/coverage_operations.py

# Check only changed lines (recommended for PRs)
python3 .claude/skills/test-coverage/scripts/coverage_operations.py --diff-only --threshold 70

# Post report to PR
python3 .claude/skills/test-coverage/scripts/coverage_operations.py --diff-only --pr-comment

# Enforce threshold (exit 1 if below 80%)
python3 .claude/skills/test-coverage/scripts/coverage_operations.py --threshold 80 --enforce
```

## Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--threshold N` | 70 | Coverage threshold percentage |
| `--diff-only` | false | Only check coverage for changed lines (not full codebase) |
| `--pr-comment` | false | Post coverage report as PR comment |
| `--enforce` | false | Exit 1 if threshold not met (for CI integration) |
| `--dry-run` | false | Preview without executing |

## Supported Test Frameworks

### JavaScript/TypeScript

**Jest**:
- Detection: `jest` in `package.json` `devDependencies`
- Command: `npm test -- --coverage --coverageReporters=json-summary --coverageReporters=text`
- Report: `coverage/coverage-summary.json`

**Vitest**:
- Detection: `vitest` in `package.json` `devDependencies`
- Command: `npx vitest run --coverage --coverage.reporter=json-summary --coverage.reporter=text`
- Report: `coverage/coverage-summary.json`

### Python

**pytest**:
- Detection: `pyproject.toml` or `setup.py` exists
- Command: `uv run pytest --cov=. --cov-report=json --cov-report=term` (or `pytest` if uv unavailable)
- Report: `coverage.json`

### Go

**go test**:
- Detection: `go.mod` exists
- Command: `go test ./... -coverprofile=coverage.out`
- Report: `coverage.out`

## Workflow

The skill executes these operations in sequence (fail-fast):

1. **Detect repository type** - Identifies test framework from repo files
2. **Run coverage** - Executes appropriate coverage command
3. **Parse coverage report** - Extracts coverage metrics from JSON/text
4. **Get PR diff** (if `--diff-only`) - Fetches changed files/lines via `gh`/`glab`
5. **Calculate diff coverage** - Intersects coverage data with changed lines
6. **Check threshold** - Compares against required threshold
7. **Generate report** - Formats human-readable coverage summary
8. **Post PR comment** (if `--pr-comment`) - Adds coverage report to PR
9. **Store metrics** - Saves to memory for trend analysis

## Error Handling

| Scenario | Behavior |
|----------|----------|
| No coverage tool detected | Returns SKIPPED, suggests installing jest/pytest |
| Coverage command fails | Returns FAILED with stderr output |
| Report file missing | Returns FAILED, check command output |
| No PR found (diff-only mode) | Returns SKIPPED, uses full coverage instead |
| Malformed coverage JSON | Returns FAILED with parse error |
| Empty diff (no changes) | Returns SUCCESS with 100% diff coverage |
| Network timeout (PR fetch) | Returns FAILED after 30s timeout |
| Memory server unavailable | Skips memory storage, continues workflow |

## Troubleshooting

**Coverage command fails**:
- Check that test framework is installed (`npm install` or `uv sync`)
- Verify tests run successfully first (`npm test` or `pytest`)
- Check for syntax errors in test files

**Report file not found**:
- Ensure coverage command generates the expected report file
- Check working directory is repo root
- Verify coverage reporters are configured correctly

**PR diff not found**:
- Ensure you're on a PR branch (e.g., `bot/RHCLOUD-12345`)
- Check `gh` CLI is authenticated (`gh auth status`)
- Verify PR has been created already

**Threshold not met**:
- Add tests for uncovered lines
- Run coverage again to verify improvement
- Check uncovered lines in report output
